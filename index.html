More About Me

I am an engineer with a diverse background beginning with my education at Southern Illinois University (SIUE). I earned a bachelor's degree in Mechanical Engineering at SIUE, and began working at an electric motor company called Nidec Motor Corporation. I was part of the team working for their largest customer with the highest demand and pressure on us. We worked heavily with the customer to research and develop new modern solutions for their problems, then bring these ideas to life through extensive testing and review. I then made a slight change and went into a sales engineering role at Ametek O'Brien Corporation. There I started as customer facing and focused on communication and relationships which was a skill I saw was valuable and I wanted to improve on. I communicate with the customer, collect the necessary information to fully understand their problem at hand and enter this into application engineering to design a custom solution. I would then provide this to the customer explaining any details associated, then receive their PO and process it. Of course, this came with all of the customer facing responsibilities like if troubleshooting, problem resolution, and cost mitigation. I then became more of a utility engineer as I was still doing the customer facing role, but I also learned how to design these special or custom solutions so I was a able to provide my customers with everything needed without needing to get other engineers involved. I also covered the largest area at this point so I certainly gained skills on organizing, prioritizing, and efficiency.

I did have some background in programming through school and of course it interested me but I didn't have the knowledge of myself at the time to know I should pursue it. A few years back I pursued an interest in Python and I came up with projects in discord.py and did some other small projects but it just wasn't as interesting without having a real compelling problem to solve. A close friend of mine knew of my background and asked if I would be interested in creating some automation and integrating webhooks and APIs for his membership group. I took him up on his offer and we've been working together since 2022. I focused on Python primarily with added external libraries to accomplish the automation and connection objectives, but moved on to other skills as well as demand for them became apparant. Once I became confident with Python, I spent time learning differences between Python and other languages like JavaScript and C++. We continued developing software aspects of the group and eventually got into upgrading and optimizing the website. This offered an opportunity to learn and apply HTML and CSS on a real use case. It was terrific experience, and I'm proud of our work on it. It has since been professionally redeveloped and unfortunately, I have to admit he did do a better job! But it was really awesome working alongside a professional website developer and learning from him!

This is where my real software development started. I kept digging into the group and optimizing things, improving the aesthetics of everything, adding webhooks and value add in places, and so on, but this is where I started getting into real development. We had a very weak pipeline for our members. It was all manual and had many holes where people could get through the paywall. Security and access was a mess and again it was all manual. As things grew it only got messier and harder to maintain. In response, I created our software pipeline to collect user given data and verify this alongside the paid account to grant access. The tricky part of this is multiple verification steps need to happen because discord usernames are not personal names. I created a system to optimize this and drastically increase the security of database. Using multiple API connections the process is almost completely automated now and I am looking to fully automate it very soon. It is very close. This pushed me further into software engineering and eventually into data engineering where I learned how to manipulate and handle data alongside PII (Personal Identifying information). I started working on personal projects centered around true data pipelines.

I started my data engineering journey by taking the Google Cloud Professional Data Engineering Course which allowed me to work closely with GCP (Google Cloud Platform), BigQuery, DataProc, DataFlow, and numerous other tools. I then took the Machine Learning Specialization course by DeepLearning.AI and I began working on projects to build these skills.

Key Projects

My New York taxi fare project is a machine learning project creating a model that could successfully predict the taxi fares given historical data which is was pulled from a public database. This required cleaning, and manipulating data into usable tables then using the new data to create models. Multiple models were created and tested to figure out the caveats and take outliers into consideration. I could have just removed the outliers because there were some extra variables like JFK static rate fares, but how can a good model just ignore areas around airports? This was an awesome project. I love ML and would love to apply more ML models to my work.

My Youtube Analytics project is a data assessment project meant to practice creating user-friendly visual dashboards specifically designed for non-data engineers or people without data manipulation and assessment experience. I already had some experience with Power BI before using Looker, which is the dashboard I used for this project, but this was also a super interesting project. There are tons of tools and visuals that can be created and manipulated on the chart. So much so, it becomes difficult to keep everything on a single page. That being said, my priority for this dashboard was to be user friendly, so a single page where all the data is collected and manipulatable using visual filters was very important.

My next project will be a project centered around creating a full data pipeline, likely using DataProc and other GCP tools.
